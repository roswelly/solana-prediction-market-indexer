What’s in the box

Five packages, one mental model: events in, state out, strategies act on state.

◆On-chain AMM program

Anchor/Rust binary-outcome CPMM with oracle resolution, configurable fees, slippage & math-overflow guards, and four typed events consumed by the indexer.

⇢Chain indexer

Pluggable ingestion — RPC logsSubscribe today, Yellowstone Geyser gRPC behind the same interface. Per-market ordered pipeline, idempotent upserts, monotonic cursor.

⚡REST + WebSocket API

Fastify service: markets, trades, OHLCV candles, global + per-trader stats. Live /ws feed fanned out from a Redis pub/sub channel.

◈Market-maker bot

Symmetric bid/ask quotes around short-window fair value with tanh-bounded inventory skew. Size scales with reserve depth.

∑Arbitrage bot

Two signals: sum-of-prices dislocation (pYes + pNo < 1) and stale-mid vs. 30s VWAP. Only fires when post-fee edge clears the configured floor.

🛡Risk + execution

Pre-trade risk manager with per-market position cap and rolling daily loss limit. Executor with clean dry-run / live seam for your RPC + priority-fee stack.

📈Replay backtester

Streams historical trades out of Postgres to tune strategy parameters without hitting the chain.

⚙Ops-ready

Multi-stage Dockerfile, docker-compose, GitHub Actions CI (TS + anchor build), Prometheus metrics, structured pino logs, Zod-validated env.

All endpoints are versioned implicitly; everything returns JSON. WebSocket uses a simple { op, channels } protocol.

Trading automation

Two strategies behind a common Strategy interface, sharing a pre-trade risk manager and an idempotent executor.

MMMarket maker

Fair value = rolling 5-minute trade-weighted mean. Bid = fv − s − skew, ask = fv + s − skew, where skew is a tanh-bounded function of current inventory. Quote size is 1% of the thinner reserve, capped by the risk manager.

ARBStatistical arbitrage

Signal A: sum-of-prices < 1 after fees → probe both outcomes. Signal B: 30s VWAP vs. AMM mid deviation > BOT_ARB_MIN_EDGE_BPS → take the lagging side.

✓Pre-trade risk

Per-market USD position cap, rolling daily loss limit, idempotency keyed on clientId. Rejects are logged with reason and never reach the wire.

Quick start

Zero-to-live-indexer in about two minutes on a laptop.

# 1. Install dependencies
npm install

# 2. Bring up Postgres + Redis, apply schema, seed synthetic data
cp .env.example .env
docker compose up -d postgres redis
npm run db:migrate
npm run db:seed

# 3. Run off-chain services
npm run indexer:dev   # ingests from devnet by default
npm run api:dev       # http://localhost:4000
npm run bot:dev       # dry-run until BOT_ENABLED=true

# 4. Build & test the Anchor program
anchor build
anchor test
